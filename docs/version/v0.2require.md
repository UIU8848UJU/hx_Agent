# HX Agent v0.2 技术说明（MVP）

## 1. 项目目标
HX Agent 是一个本地知识库 Agent，面向“学习问答 + 笔记沉淀 + 自动整理”场景。  
核心原则：SQLite-first、CLI-first、可演进到 LLM/向量检索。

## 2. 当前能力
- 文档入库：`ingest`（md/txt，增量更新）
- 检索：`search`（SQLite FTS5）
- 问答：`ask`（规则摘要）
- 陪伴学习：`study start/ask/end/notes`
- 自动整理：`organize`
- 笔记重排：`reformat`

## 3. 模块结构
- CLI 入口：`hx_agent/cli.py`
- 入库与切块：`hx_agent/ingest/*`
- 存储层：`hx_agent/index/meta_store.py`
- 规则问答：`hx_agent/ask.py`
- 学习会话服务：`hx_agent/study/service.py`
- 整理服务：`hx_agent/organize/service.py`
- LLM 骨架：`hx_agent/llm/client.py`
- 数据库结构：`hx_agent/index/sql/schema.sql`

## 4. 数据流程
1. `hx-agent ingest data`
2. 扫描文件 -> 切块 -> 写入 `files/chunks/chunks_fts`
3. `study ask` 时：
   - 按会话 `source_path` 做范围检索
   - 命中则生成回答并存消息/笔记
   - 无命中则在同范围做兜底摘要
4. 输出回答 + 引用（path + line range + chunk ids）

## 5. 数据库表（v0.2增量）
- 既有：
  - `files`
  - `chunks`
  - `chunks_fts`
  - `runs`
- 新增：
  - `study_sessions`
  - `study_messages`
  - `study_notes`
  - `organize_runs`

## 6. 关键命令
- 初始化：`hx-agent init-db`
- 入库：`hx-agent ingest data`
- 会话开始：`hx-agent study start --name "同步机制" --source data`
- 会话提问：`hx-agent study ask "这里讲了什么核心概念？"`
- 查看笔记：`hx-agent study notes`
- 结束会话：`hx-agent study end`
- 自动整理：`hx-agent organize data --template sop`

## 7. LLM 接入状态
当前 `LLMClient` 为骨架实现，未接真实 provider。  
行为是：优先尝试 LLM（若配置），否则走规则摘要 fallback。  
环境变量预留：
- `HX_AGENT_LLM_PROVIDER`
- `HX_AGENT_LLM_MODEL`
- `HX_AGENT_LLM_API_KEY`

## 8. 已知限制
- FTS 是关键词检索，不是语义向量检索。
- “这里/这个”类指代问题依赖范围与兜底，不等于真正多轮推理。
- `source` 目前用于检索范围，不自动触发局部 ingest。
- 真实模型回答质量取决于后续 LLM 接入与 prompt 设计。
